# ADR 004 - Uso de Microservicios para Arquitectura Independiente

**Estado:** Aprobado

## Contexto
El sistema YAGNI debe escalar y manejar de manera eficiente distintos componentes funcionales, como gestión de usuarios, cocinas, logística, y refrigeradores, entre otros. Para garantizar flexibilidad, escalabilidad, y mantenibilidad, se ha evaluado la implementación de una arquitectura de microservicios. Cada componente deberá poder evolucionar de manera independiente sin afectar el resto del sistema.

## Decisión
Se ha decidido adoptar una arquitectura basada en microservicios. Cada servicio será independiente, responsable de una funcionalidad específica del sistema (por ejemplo, gestión de clientes y sus ordenes, manejo de cocinas, logística, etc.), y podrá desarrollarse, desplegarse y escalarse por separado. Los servicios se comunicarán entre sí de manera asíncrona a través de eventos y colas de mensajes, y de manera síncrona usando el API Gateway para solicitudes directas.

## Atributos de Calidad Satisfechos
- **Escalabilidad:** Permite que cada servicio escale de manera independiente, de acuerdo con la carga que maneje.
- **Modificabilidad:** Cada servicio puede ser actualizado o reparado sin afectar el resto del sistema, facilitando el mantenimiento.
- **Disponibilidad:** Facilita la redundancia de servicios individuales, mejorando la disponibilidad del sistema en general.
- **Deployabilidad:** Los equipos pueden trabajar y desplegar servicios en paralelo, reduciendo el tiempo de desarrollo y pruebas.
- **Interoperabilidad**: Cada microservicio está diseñado para gestionar una función específica. Los servicios interactúan entre sí a través de interfaces bien definidas, sin depender de la implementación interna de otros servicios.

## Ventajas
- Los servicios son independientes, lo que permite una mayor flexibilidad y capacidad de escalar.
- Cada servicio puede ser desarrollado y mantenido por equipos distintos, lo que acelera los tiempos de entrega.
- Reducción de impactos por fallas: si un servicio falla, no afecta al resto del sistema.

## Desventajas
- Aumenta la complejidad operativa, ya que se deben gestionar múltiples servicios y sus interdependencias.
- La comunicación entre servicios puede generar latencia y problemas de consistencia si no se gestionan correctamente las dependencias.
- Requiere una infraestructura más compleja para manejar el despliegue, monitoreo y balanceo de los servicios.
