# ADR 005 - Uso de API Gateway como Gatekeeper

**Estado:** Aprobado

## Contexto
El sistema YAGNI debe manejar de manera eficiente la autenticación, autorización, y distribución de solicitudes hacia varios servicios. Para lograr esto, es fundamental tener un punto centralizado para gestionar estas operaciones, protegiendo los servicios internos de accesos no autorizados y asegurando un enrutamiento adecuado de las solicitudes. Dado que el sistema utilizará una arquitectura de servicios, es necesario implementar un mecanismo de control para todas las peticiones entrantes y salientes hacia los servicios.

## Decisión
Se ha decidido utilizar un API Gateway como Gatekeeper. El API Gateway será responsable de recibir todas las solicitudes, manejar la autenticación y autorización de los usuarios, y enrutar las solicitudes hacia los servicios correspondientes. 

## Atributos de Calidad Satisfechos
- **Seguridad:** El API Gateway asegura que todas las peticiones sean autenticadas y autorizadas antes de acceder a los servicios.
- **Escalabilidad:** Facilita la escalabilidad al manejar múltiples servicios desde un único punto de entrada.
- **Modificabilidad:** Centraliza el manejo de políticas de seguridad, facilitando la gestión y el mantenimiento de estas.

## Ventajas
- Centraliza el acceso a los servicios y asegura el control de la autenticación y autorización interactuando con el Identity Provider.
- Simplifica el enrutamiento de solicitudes y permite aplicar políticas de seguridad a nivel global.

## Desventajas
- Introduce un único punto de falla en el sistema, que requiere alta disponibilidad y robustez.
- Puede añadir latencia si no se optimizan correctamente las configuraciones de enrutamiento.
