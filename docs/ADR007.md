# ADR 007 - Uso de Docker junto con los Microservicios de YAGNI

**Estado**: Propuesto.

## Contexto:

El sistema YAGNI se basa mayoritariamente en una arquitectura de microservicios, donde cada componente es independiente y se comunica con otros servicios a través de APIs. Para asegurar una implementación consistente y eficiente en diferentes entornos (desarrollo, testing, producción), se ha identificado la necesidad de un mecanismo que permita el empaquetado, despliegue y escalabilidad de estos microservicios de forma autónoma y predecible.

Docker es una plataforma de contenedorización que permite empaquetar cada microservicio con todas sus dependencias en una imagen estándar, lo que garantiza la consistencia de los entornos, la portabilidad entre máquinas y la facilidad de escalado.

## Decisión:

Se ha decidido utilizar Docker como la solución para la contenedorización de los microservicios de YAGNI. Cada microservicio se empaquetará en su propio contenedor, incluyendo sus dependencias específicas, y se gestionará utilizando Docker para garantizar un despliegue homogéneo en cualquier entorno.

## Atributos de Calidad Satisfechos:
  
- **Escalabilidad**: Docker permite la fácil replicación y escalado de los microservicios mediante orquestadores como Docker Compose o Kubernetes, mejorando la capacidad de respuesta del sistema a altas cargas.

- **Disponibilidad**: El uso de Docker junto con herramientas de orquestación permitirá desplegar microservicios de manera rápida y minimizar el tiempo de inactividad durante actualizaciones o fallos. Por otro lado, Docker asegura que cada microservicio se ejecute de manera consistente en diferentes entornos (local, staging, producción), lo que elimina problemas relacionados con dependencias o configuraciones inconsistentes.

- **Modificabilidad/Testeabilidad**: Docker simplifica el mantenimiento, ya que los microservicios son independientes y pueden ser actualizados, testeados o reparados sin afectar a otros componentes.

- **Deployabilidad**: Docker permite automatizar el proceso de construcción, despliegue y escalado, reduciendo el esfuerzo manual requerido para gestionar los servicios.

## Ventajas:

- **Aislamiento**: Cada microservicio tiene su propio contenedor con todas sus dependencias, evitando conflictos entre versiones de librerías o configuraciones.
- **Control de versiones**: Las imágenes de Docker pueden versionarse, lo que facilita la gestión de diferentes versiones de microservicios.
- **Menor dependencia de infraestructura**: Docker reduce la necesidad de preocuparse por la infraestructura subyacente, ya que los contenedores se ejecutan de manera uniforme en cualquier entorno compatible.

## Desventajas:

- **Sobrecarga inicial**: La implementación de Docker requiere la creación y mantenimiento de Dockerfiles y configuraciones adicionales, lo que puede aumentar la complejidad del proyecto inicialmente.
- **Recursos adicionales**: Aunque los contenedores son ligeros, ejecutar múltiples instancias de Docker en un solo host puede consumir más recursos que ejecutar los servicios directamente.
- **Curva de aprendizaje**: El equipo debe familiarizarse con Docker y su ecosistema (Docker Compose, orquestadores, etc.), lo que puede requerir tiempo y capacitación.
