# ADR 001 - Uso de Sequelize como ORM

**Estado**: Aprobado

## Contexto
Se busca establecer un mecanismo eficiente para manejar la persistencia de datos en la plataforma YAGNI, que incluye el registro de clientes, pedidos, cocinas, locales y refrigeradores. La solución debe ser escalable, segura y fácil de mantener, teniendo en cuenta la integración con MySQL.

## Decisión
Se ha decidido utilizar **Sequelize** como el ORM (Object-Relational Mapping) para interactuar con las bases de datos MySQL. Sequelize ofrece una interfaz intuitiva y poderosa para definir modelos, realizar consultas y manejar la migración de esquemas de manera sencilla.

## Atributos de Calidad Satisfechos
1. **Modificabilidad**: La utilización de Sequelize permite una estructura de código más limpia y fácil de entender, lo que facilita el mantenimiento y la adición de nuevas funcionalidades.
2. **Escalabilidad**: Sequelize es compatible con múltiples bases de datos, lo que permite escalar el sistema a diferentes entornos de producción y desarrollo.
3. **Seguridad**: Sequelize ayuda a prevenir inyecciones SQL a través de su uso de consultas parametrizadas.
4. **Usabilidad**: Proporciona una API fácil de usar que reduce la curva de aprendizaje para los desarrolladores.

## Ventajas
- Facilita la interacción con la base de datos sin necesidad de escribir consultas SQL complejas.
- Proporciona características como validaciones, asociaciones y migraciones que mejoran la calidad del código.
- Compatible con múltiples bases de datos, lo que permite flexibilidad en la elección del sistema de gestión de bases de datos.

## Desventajas
- Puede introducir una capa de abstracción que ocasionalmente podría afectar el rendimiento en consultas complejas.
- Requiere un tiempo de aprendizaje para desarrolladores que no están familiarizados con ORM.
